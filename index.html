Description:

        Our project is a suite of various arcade games using the OLED screen, the TV remote as an input device, and the onboard accelerometer as another input device for some of the games. We will also be using AWS to have the user’s initials and scores emailed to them.

Design:

Functional Specification State Machine Diagram: Users will first be able to pick a game from the selection screen. Once a game has been selected, it will load onto the screen. Users will be able to play the game to try and aim for a high score. Once they get a game over, the game will end and they will type in their initials. Their score and initials will be sent to their email and the user will be led to the game selection screen again.

System Architecture Block Diagram: Users will be able to pick which game they want to play using the TV remote input at the selection screen on the OLED. If they choose an accelerometer game, the accelerometer will be used as their controller input, if they choose a TV remote game, the TV remote will be used as their controller input. Once their game ends, they will type their initials using the TV remote. AWS will email the user their initials and score.

Implementation:

To connect to the OLED, we used the write command and data and adafruit initialization functions. The UI, selection screens, game, and gameover screens are shown and updated based on inputs on the OLED. Users can use the arrow buttons, OK button, and menu button to navigate through the UI.

To obtain the values from the CC3200 accelerometer, we used I2C read and write. Based on these values, they controlled the accelerometer games “Dodgeball” and “Baskets”. For “Dodgeball”, the accelerometer was used to control the position of the player in order to dodge objects coming from the edges of the screen. If the x and y values of the player and objects are equal to each other, the game is over. For “Baskets”, the accelerometer was used to control the x-position of the player in order to collect points and dodge objects coming from the top of the screen. If the x and y values of the player and points are equal to each other, the score is increased. For the player and objects, the game is over. For both games, we used a for loop and an array to create many objects and to store their properties such as speed.

Challenges:

        A challenge we faced was getting the game run smoothly. Drawing objects on the OLED takes time and with many objects on screen, this causes the game to run slowly. In order to combat this, we used smaller objects and only redrew objects that needed to be redrawn. For the snake game, only the front and back of the snake was redrawn. For the accelerometer games, the objects were made smaller in order for the game to run smoothly.

Future Work:

        In the future, we would like to make the games run smoother. Updating each object on screen takes some time which limits the amount of objects and the size of objects in order to make everything run smoothly. We would like to add a website that keeps track of initials and highscores so users can view their and others scores. We would also like to add even more games so our project can feel like an actual arcade with many options to select.
